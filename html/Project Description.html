Technology Stack

Description

The aim of this project is to create an end-to-end big data pipeline. The stages of the pipeline are as follows:

  1. Data Collection and Ingestion
    • Building a relational database using PostgreSQL and importing relational data to HDFS using Sqoop.
  2. Data Storage/Preparation
    • Creating Hive Tables with compressed file format (AVRO).
      • Storing the data in the data warehouse Hive for analytics.
  3. Data Analysis
    • Perform Exploratory Data Analysis (EDA) with HiveQL on Tez engine.
    • Perform Predictive Data Analysis (PDA) via building ML models using SparkML.
      • This part also includes feature extraction and preprocessing data.
  4. Presentation
    • Presenting the analysis results in a dashboard using Streamlit.

The deliverables of this project are:

  1. The project repository
    • You have to use only Python programming language in addition to SQL and HiveQL for EDA and data preparation.
    • You have to use the project repository template attached to this tutorial for your project. Do not fork but duplicate it and start working on your project.
      • The repository README.MD includes the rules of using the template.
      • The scripts should generate reproducible results. For instance, If I run the script stage1.sh for the second time, it should not give me any errors, so you should make sure that you cleared the objects before creating new ones.
    • For EDA part
      • You need to analyze the data and display at least 5 different insights in the dashboard.
      • For each insight drawn from the data, you need to create charts and add a description for it.
    • For ML part
      • You need to build two different kinds of models like Random Forest and Naive Bayes for classification task.
      • For data split, use 70% training and 30% test.
      • Fine-tune the hyper-parameters via grid search on 3 different values for each hyperparameter. It is enough tp optimize for only 2 hyperparameters. The number of epochs/iterations is not considered as a hyperparameter.
      • You should use the following performance metrics (reference):
        • Accuracy and F1 for multiclass classification tasks
          • Area Under ROC and Area Under PR for binary classification tasks.
        • RMSE and R2R2 for regression tasks.
        • Mean Average Precision and NDCG for recommendation tasks.
      • Perform k-fold cross validation (where k=4) and select the best model.
      • Show the performance of both best models of each kind in the dashboard.
    • For the dashboard.
      • It is enough to create a single static dashboard via Streamlit.
      • You have to display the data characteristics such as number of data instances, number of features, feature names,…
      • You need to display all the results of the analytics.
      • Check the grading criteria to see the requirements needed to be fulfilled.
      • The dashboard does not have to be interactive.
  2. The presentation
    • You should explain the goal of your project, for instance you are trying to predict ratings of users to build a good recommendation system. Expand this sentence, add some figures and images to explain the objective.
    • The dataset characteristics.
    • How you analyzed the data.
    • What challenges you encountered.
    • A short demo you present the dashboard in Streamlit and demonstrate the results you obtained.

If your dataset is large, you can divide it into many parts and store them in Github.

Dataset Criteria

  1. From the perspective of relational databases, the database should include at least two entities and one relationship between them.
  2. Data is big in terms of volume.
    • At least 20,000 data samples in total entities and more than 100MB.
  3. At least 7 explanatory variables (features)
  4. Either datetime or geospatial features must be present in the data (both can be).
  5. The goal of predictive analysis with regard to the data should be detemined (Regression, Classification, or Collaborative filtering).
  6. The dataset should not be one of the datasets you worked on during the labs or assignments.
  7. Preferred format of the dataset files is “csv”.
  8. JSON and Excel files are fine but you need to convert them to csv.
  9. The TA needs to verify your dataset
  10. You should find a dataset before March 31.

Team Formation Criteria

  1. Each teammate should participate in the project and presentation.
  2. You have to work in teams of two people.
    • For exceptional situations, you need to contact the TA.
  3. You should select a team before March 31.

Grading Criteria

  • [15 points] Stage I
    • [5 points] You built all tables in the database. You added all constraints if any.
    • [5 points] You loaded the data to the tables.
    • [5 points] You imported the data to HDFS via Sqoop.
  • [5 points] Stage II
    • [5 points] You created Hive Tables and stored the data.
  • [40 points] Stage III
    • [10 points] EDA
      • [2 points] for each insight provided.
    • [30 points] PDA
      • [10 points] for the first model.
      • [10 points] for the second model.
      • [5 points] for cross validation and hyperparameter tuning.
      • [5 points] for prediction of a specific data sample.
  • [20 points] Stage IV
    • [20 points] for the dashboard.
      • [2 points] for showing the data characteristics.
      • [2 points] for each insight with provided description.
      • [3 point] for showing the performance of models.
      • [3 points] for prediction results.
      • [2 points] for visual quality of dashboard.
  • [15 points] Project Defence
    • [5 points] for presentation
    • [10 points] for answering the questions.
  • [5 points] Check code quality using pylint
    • [4 points] if only 2-3 warnings raised
    • [3 points] if only 4 warnings raised
    • [2 points] if only 5-6 warnings raised
    • [1 points] if 7+ warnings raised

The final grade of the project is computed as follows:
final grade=round(40∗total points100)final grade=round(40∗total points100)

Possible cases for penalties:

  • You did not follow the repo structure or did not follow the rules in the repostiory.
    • Remove the points assigned for code quality
    • Add penalty [-30%] for each other grading criterion.
  • You did not attend the defence.
    • Remove the points assigned for project defence
    • Add penalty [-50%] for each other grading criterion.
  • Late submissions.
    • Add penalty [-50 points].
    • If the student submitted after the defence day, then their submission is not considered.
